# 智能诊断流程

> 流程编号：FLOW-03-05 | 版本：v1.1 | 更新时间：2026-06-13

**流程说明**：用户描述故障后，系统结合维修手册、案例、车辆信息进行综合诊断，输出初步结论和风险等级建议。

---

## 完整智能诊断流程图

```mermaid
flowchart TD
    A[用户进入智能诊断] --> B[选择车辆]
    B --> C[输入故障描述]
    C --> D[可选上传图片]
    D --> E[可选输入故障码]

    E --> F[提取关键信息]
    F --> G{信息是否充分}
    G -- 否 --> H[追问补充信息]
    H --> I[用户补充]
    I --> F

    G -- 是 --> J[查询车辆档案]
    J --> K[六路混合检索]
    K --> L[综合生成初步诊断]
    L --> M[判断风险等级]

    M --> N{风险等级}
    N -- 低风险 --> O1[提供自查步骤]
    N -- 中风险 --> O2[建议预约服务站]
    N -- 高风险 --> O3[建议停止使用并尽快报修]
    N -- 紧急 --> O4[建议立即停车并呼叫救援]

    O1 --> P[展示诊断结果和引用来源]
    O2 --> P
    O3 --> P
    O4 --> P

    P --> Q{用户下一步}
    Q -- 继续观察 --> R[记录诊断会话]
    Q -- 需要报修 --> S[进入智能报修流程]
    Q -- 继续追问 --> T[进入多轮对话]
```

---

## 风险等级规则示意

| 风险等级 | 触发条件示例 | 建议动作 |
|---|---|---|
| 低风险 | 异响但不影响驾驶 | 自查 + 保养时检查 |
| 中风险 | 动力轻微下降、偶发故障灯 | 近期预约服务站 |
| 高风险 | 无法启动、制动异常、电池告警 | 停止使用，立即报修 |
| 紧急 | 电池热失控、制动失灵、冒烟 | 立即停车，拨打救援 |

---

## 诊断输出建议结构

1. 风险等级
2. 可能原因列表
3. 建议动作
4. 是否可继续使用
5. 引用来源
