# 多轮对话上下文流程

> 流程编号：FLOW-03-04 | 版本：v1.0 | 更新时间：2026-06-12

**流程说明**：用户在售后问答中经常连续追问，系统需要保留上下文，识别指代关系，避免每轮都从零开始检索。

---

## 多轮对话流程图

```mermaid
flowchart TD
    A[用户发起第一轮问题] --> B[创建会话 qa_session]
    B --> C[记录问题到 qa_messages]
    C --> D[意图识别 + 实体抽取]
    D --> E[检索 + 生成答案]
    E --> F[保存本轮上下文 问题/答案/实体/意图]

    F --> G{用户是否继续追问}
    G -- 否 --> H[🔚 会话结束]
    G -- 是 --> I[接收下一轮问题]

    I --> J[判断是否依赖历史上下文]

    J --> K{问题类型}
    K -- 独立问题 与历史无关 --> L[直接处理当前问题]
    K -- 指代问题 涉及"这个""它""刚才说的" --> M[指代消解]
    K -- 追问问题 继续上一话题 --> N[融合历史上下文]
    K -- 转换话题 明确切换 --> O[重置上下文 处理新话题]

    M --> P[结合历史改写当前问题]
    N --> P

    P --> Q[重新检索]
    L --> Q
    O --> Q

    Q --> R[生成新答案]
    R --> S[更新会话上下文]
    S --> G
```

## 典型多轮对话示例

### 示例一：指代消解

```
用户: T5轻卡动力电池质保几年？
系统: 根据质保手册，T5轻卡动力电池质保期为5年或20万公里……

用户: 那超过里程但没超时间呢？
# 系统识别"那"指代上一轮的"动力电池质保"
# 改写问题为："T5轻卡动力电池超过质保里程但未超质保年限时如何处理"
# 重新检索并回答
```

### 示例二：追问深入

```
用户: 6万公里需要做哪些保养？
系统: 60000公里需要更换减速器润滑油、检查制动系统……

用户: 其中换润滑油大概多少费用？
# 系统识别"其中换润滑油"关联上轮的"更换减速器润滑油"
# 继续在保养费用知识库中检索
```

### 示例三：转换话题

```
用户: 车辆无法启动怎么处理？
系统: 建议检查低压电瓶、高压上电条件……

用户: 好的，另外我想问一下保养周期
# 系统识别"另外"为话题切换信号
# 重置本轮上下文，独立处理保养咨询
```

## 上下文窗口管理策略

| 策略 | 说明 |
|---|---|
| 保留最近 5 轮 | 超过 5 轮的历史压缩摘要 |
| 关键实体持久化 | 车型、VIN、里程、故障码在整个会话中保持 |
| 话题切换重置 | 检测到明确的话题切换词时，重置检索上下文 |
| 超时自动结束 | 会话 30 分钟无活动自动标记为结束 |

```python
def build_session_context(session_messages: list, max_rounds: int = 5) -> str:
    recent = session_messages[-max_rounds:]
    context_parts = []
    for msg in recent:
        context_parts.append(f"用户: {msg['question']}")
        context_parts.append(f"助手: {msg['answer'][:200]}...")  # 截断避免超长
    return "\n".join(context_parts)
```
