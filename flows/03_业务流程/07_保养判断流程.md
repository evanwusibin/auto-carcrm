# 保养判断流程

> 流程编号：FLOW-03-07 | 版本：v1.1 | 更新时间：2026-06-13

---

## 完整流程图

```mermaid
flowchart TD
    A[用户询问是否需要保养] --> B[识别保养咨询意图]
    B --> C{是否已绑定车辆}
    C -- 否 --> D[引导绑定车辆]
    D --> E[获取车辆信息]
    C -- 是 --> E

    E --> F[检索保养手册]
    F --> G[查询历史保养记录]
    G --> H[计算保养状态]
    H --> I{是否需要保养}

    I -- 已超保养节点 --> J1[提示尽快预约保养]
    I -- 接近保养节点 --> J2[提示提前预约]
    I -- 保养状态正常 --> J3[提示下次建议保养时间或里程]
    I -- 信息不足 --> J4[提示补充当前里程]

    J1 --> K[展示本次建议保养项目]
    J2 --> K
    J3 --> L[展示下次保养节点]
    J4 --> M[追问当前里程]
    M --> E

    K --> N{用户是否预约}
    N -- 是 --> O[进入预约或报修流程]
    N -- 否 --> P[结束]
    L --> P
```

---

## 判断规则示意

| 状态 | 含义 |
|---|---|
| `overdue` | 已超过建议保养节点 |
| `due_soon` | 即将到达保养节点 |
| `normal` | 当前保养状态正常 |
| `unknown` | 信息不足，无法判断 |

---

## 核心判断依据

1. 当前里程
2. 最近一次保养里程
3. 最近一次保养时间
4. 车型对应的保养周期规则
