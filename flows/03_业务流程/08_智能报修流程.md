# 智能报修完整流程

> 流程编号：FLOW-03-08 | 版本：v1.0 | 更新时间：2026-06-12

**流程说明**：从用户确认需要报修，到经销商受理、维修处理、工单关闭的端到端完整流程，覆盖三类角色协同。

---

## 完整智能报修流程图

```mermaid
flowchart TD
    A[👤 用户确认需要报修] --> B[系统生成报修单草稿]

    B --> B1[🚗 自动带出: 用户信息]
    B --> B2[🚗 自动带出: 车辆信息<br/>VIN/车型/里程]
    B --> B3[🤖 自动带出: 故障描述<br/>来自诊断会话]
    B --> B4[🤖 自动带出: AI诊断结果<br/>可能原因+风险等级]
    B --> B5[🤖 自动带出: 质保预判结果]

    B1 --> C[用户补充报修信息]
    B2 --> C
    B3 --> C
    B4 --> C
    B5 --> C

    C --> C1[联系人/联系电话]
    C --> C2[当前所在位置]
    C --> C3[车辆是否可正常行驶]
    C --> C4[期望服务时间]
    C --> C5[可选: 上传图片/视频]
    C --> C6[选择服务站]

    C1 --> D[用户确认提交]
    C2 --> D
    C3 --> D
    C4 --> D
    C5 --> D
    C6 --> D

    D --> E[创建报修单<br/>state: submitted]
    E --> F[系统推送通知给经销商]

    F --> G[🔧 经销商/服务站接收报修单]
    G --> H[服务顾问查看报修详情]
    H --> H1[查看AI初步诊断]
    H --> H2[查看质保预判]
    H --> H3[查看车辆档案和保养记录]

    H1 --> I[服务顾问受理<br/>state: accepted]
    H2 --> I
    H3 --> I

    I --> J[联系客户确认<br/>state: contacting]
    J --> K{车辆情况}
    K -- 可以行驶 --> L[预约进站时间]
    K -- 无法行驶 --> M[安排救援/拖车]

    L --> N[车辆进站<br/>state: waiting_vehicle → inspecting]
    M --> N

    N --> O[维修技师检测车辆]
    O --> P[读取故障码]
    P --> Q[技师查询维修知识库<br/>RAG问答辅助]
    Q --> R[确认故障原因]

    R --> S{质保归属判断}
    S -- 质量问题且在保 --> T[走质保流程<br/>state: warranty_review]
    S -- 非质保/超保 --> U[生成维修报价<br/>state: quotation]
    S -- 不确定/疑难 --> V[升级厂家技术支持<br/>state: factory_support]

    T --> W[免费维修处理<br/>state: repairing]
    U --> X{客户是否确认}
    X -- 确认 --> W
    X -- 拒绝 --> Y[取消维修<br/>state: cancelled]

    V --> Z[厂家工程师远程支持]
    Z --> AA[给出维修方案]
    AA --> W

    W --> AB{是否需要等待配件}
    AB -- 是 --> AC[state: waiting_parts]
    AB -- 否 --> AD[完成维修<br/>state: completed]
    AC --> AE[配件到货] --> AD

    AD --> AF[技师填写维修结论]
    AF --> AF1[故障原因确认]
    AF --> AF2[维修方案]
    AF --> AF3[更换配件清单]
    AF --> AF4[质保最终结论]
    AF --> AF5[费用明细]

    AF1 --> AG[推送完成通知给客户]
    AG --> AH[👤 客户确认完成<br/>state: customer_confirmed]
    AH --> AI[客户评价服务]
    AI --> AJ[工单关闭<br/>state: closed]

    AJ --> AK{是否为典型案例}
    AK -- 是 --> AL[技师提交典型案例]
    AL --> AM[工程师审核案例]
    AM --> AN[案例入知识库<br/>state: published]
    AK -- 否 --> AO[🔚 流程结束]
    AN --> AO
```

## 报修单自动带出字段说明

系统自动带出的信息来源：

| 字段 | 来源 | 说明 |
|---|---|---|
| 用户信息 | 登录账号 | 姓名、手机号 |
| 车辆信息 | vehicles 集合 | VIN、车型、里程、购车日期 |
| 故障描述 | diagnosis_sessions | 用户输入的故障现象 |
| AI 诊断结果 | diagnosis_sessions | 可能原因、风险等级 |
| 质保预判 | 质保预判接口 | 时间/里程/保养状态判断 |

用户只需要补充：联系信息、位置、是否可行驶、期望时间、服务站选择。大幅减少手动填写负担。

## 报修单关键状态说明

| 状态 | 操作方 | 触发时机 |
|---|---|---|
| `submitted` | 用户提交 | 用户点击确认报修 |
| `accepted` | 服务顾问 | 服务站受理工单 |
| `inspecting` | 系统/技师 | 车辆到站开始检测 |
| `warranty_review` | 服务顾问 | 需要判断是否属于质保 |
| `repairing` | 技师 | 开始维修 |
| `completed` | 技师 | 维修完成 |
| `closed` | 系统 | 客户确认 + 评价完成 |
