## 🔄 流程节点编号规范

所有流程节点统一按以下规则编号：

| 前缀 | 所属流程 | 说明 |
|---|---|---|
| `R-` | **HTTP 路由层** | `app/api/routers/*.py`，负责参数解析、依赖注入、响应包装 |
| `P-` | **页面编排层** | `app/process/{import_,query}/page/*.py`，负责把 router 请求编列为 graph 调用 + 状态/历史/审计 |
| `G-` | **图编排层（LangGraph）** | `app/process/{import_,query}/agent/main_graph.py`，负责多节点有向无环任务调度 |
| `RAG-` | **RAG 知识入库节点** | 文档解析 / 切分 / 向量化 / 入库 / 审核发布 |
| `QA-` | **在线检索问答节点** | 意图识别 / 实体抽取 / 多路检索 / 融合 / Rerank / 生成 / 溯源 |
| `C-` | 终端客户业务节点 | 提问 / 诊断 / 报修 / 查进度 |
| `D-` | 经销商/服务站节点 | 接单 / 检测 / 维修 / 提交案例 |
| `F-` | 厂家/管理员节点 | 上传知识 / 审核发布 / 技术支持 / 分析优化 |

> **调用链路**（从外到内）：`HTTP Request → R-* (router) → P-* (page) → G-* (graph) → RAG-*/QA-* (节点) → LLM/向量库/外部服务`

## 🛠️ 如何在本地渲染 Mermaid 图

**方式一：VS Code + Mermaid Preview 插件**
安装 `Mermaid Preview` 插件，打开 `.md` 文件后点击预览即可。

**方式二：Gitee / GitHub 在线渲染**
直接在 Gitee/GitHub 仓库页面打开 `.md` 文件，Mermaid 代码块会自动渲染。

**方式三：Typora**
Typora 原生支持 Mermaid，直接打开 `.md` 文件即可看到图表。
