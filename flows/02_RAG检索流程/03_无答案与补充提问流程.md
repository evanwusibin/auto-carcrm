# 无答案与补充提问流程

> 流程编号：FLOW-02-03 | 版本：v1.0 | 更新时间：2026-06-12

**流程说明**：RAG 系统不能假装什么都能回答。当检索置信度不足时，系统应明确识别原因，引导用户补充信息或提示资料不足，而不是强行生成答案。

---

## 完整流程图

```mermaid
flowchart TD
    A[用户问题进入检索] --> B[多路混合检索]
    B --> C[计算召回结果置信度]

    C --> D{最高相似度分数判断}
    D -- 分数 >= 0.75 --> E[✅ 正常进入生成流程]
    D -- 分数 0.5~0.75 --> F[⚠️ 低置信度：谨慎生成]
    D -- 分数 < 0.5 --> G[❌ 无有效结果：分析原因]

    F --> F1[生成时在答案前加免责说明]
    F1 --> F2[提示"以下回答供参考，建议联系服务站确认"]
    F2 --> E

    G --> H{原因诊断}

    H -- 缺少车型信息 --> I1[🔁 追问: 请告诉我您的车型 如T5轻卡/T7重卡等]
    H -- 缺少VIN/车辆未绑定 --> I2[🔁 追问: 请绑定车辆或输入VIN号]
    H -- 缺少当前里程 --> I3[🔁 追问: 请告知当前行驶里程]
    H -- 缺少故障码 --> I4[🔁 追问: 仪表盘是否显示故障码或故障灯]
    H -- 问题表述不清 --> I5[🔁 追问: 能否更详细描述问题现象]
    H -- 知识库无相关资料 --> I6[📭 提示: 当前知识库未收录该内容 建议联系授权服务站]
    H -- 权限不足 --> I7[🚫 提示: 您无权查看该类型资料]
    H -- 问题超出业务范围 --> I8[↗️ 提示: 该问题超出售后服务范围 建议联系对应部门]

    I1 --> J[等待用户补充信息]
    I2 --> J
    I3 --> J
    I4 --> J
    I5 --> J

    J --> K{用户是否补充}
    K -- 已补充 --> L[重新发起检索]
    L --> B
    K -- 未补充/超时 --> M[记录为无答案问题 进入知识缺口池]

    I6 --> M
    I7 --> N[记录权限拦截日志]
    I8 --> O[记录超范围日志]

    M --> P[知识缺口分析]
    P --> Q[管理员/厂家补充知识]
    Q --> R[重新导入知识库]
```

## 追问话术设计

系统追问时，应保持自然对话感，避免生硬的"请输入X"。

| 缺失信息 | 推荐追问话术 |
|---|---|
| 车型 | "您好，为了给您更准确的解答，请问您的车辆是哪款车型呢？比如T5轻卡、T7重卡等。" |
| 里程 | "请问您的车辆目前行驶了多少公里了呢？这样我可以更准确地判断是否需要保养。" |
| 故障码 | "您的仪表盘上有没有出现故障灯或显示错误代码呢？如果有，可以告诉我代码是什么。" |
| 故障描述 | "能再描述一下具体的问题现象吗？比如是什么时候出现的、有没有异响、有没有故障灯等。" |
| 车辆未绑定 | "我还没有找到您的车辆信息，请问可以提供一下VIN码（车架号）吗？通常在驾驶证或车辆铭牌上可以找到。" |

```json
{
  "question": "充电桩故障和车辆故障怎么区分",
  "intent_type": "fault_diagnosis",
  "no_answer_reason": "knowledge_gap",
  "retrieval_max_score": 0.38,
  "user_id": "U10001",
  "vehicle_model": "T5",
  "created_at": "2026-06-12T10:30:00",
  "handled": false,
  "suggested_action": "补充充电系统故障排查与充电桩故障区分知识"
}
```

## 知识缺口分类

| 类型 | 描述 | 建议处理 |
|---|---|---|
| `knowledge_gap` | 知识库未收录该内容 | 厂家补充相关手册或FAQ |
| `chunk_split_issue` | 知识存在但切分不当导致检索失败 | 优化切分策略并重新索引 |
| `embedding_mismatch` | 语义差距太大 | 考虑换用更好的 Embedding 模型或增加同义词扩展 |
| `permission_block` | 权限拦截 | 检查权限配置是否合理 |
| `out_of_scope` | 超出业务范围 | 明确边界，如有高频则扩展范围 |
