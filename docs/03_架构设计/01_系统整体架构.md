## 一、架构总览

                           │

## 二、核心链路说明

### 2.1 知识入库链路（离线）

```
原始文档上传
    ↓
文件格式识别（PDF / Word / Excel / 图片扫描件）
    ↓
内容解析（PyMuPDF / python-docx / pandas / OCR）
    ↓
文本清洗（去页眉页脚 / 去噪 / 合并断行 / 表格转文本）
    ↓
结构识别（章节标题 / 故障码 / 保养里程 / 质保期限）
    ↓
Chunk 切分（按章节 / FAQ对 / 案例 / 固定长度兜底）
    ↓
元数据绑定（车型 / 部件 / 版本 / 生效时间 / 权限）
    ↓
Embedding 向量化（bge-large-zh / bge-m3）
    ↓
写入 MongoDB（文本 + 向量 + 元数据）
    ↓
建立向量索引（MongoDB Atlas Vector Search）
    ↓
审核 → 发布 → 可被检索
```

### 2.2 在线问答链路

```
用户自然语言提问
    ↓
问题预处理（清洗 / 上下文补全 / 多轮改写）
    ↓
意图识别（保养 / 质保 / 故障 / 案例 / 流程 / 报修）
    ↓
实体抽取（车型 / VIN / 故障码 / 里程 / 部件）
    ↓
多路混合检索
    ├── 向量语义检索（MongoDB Vector Search）
    ├── 关键词精确检索（BM25）
    ├── 结构化业务查询（车辆档案 / 保养记录 / 质保规则）
    └── 历史案例检索
    ↓
元数据过滤（车型 / 版本 / 有效期 / 权限）
    ↓
结果融合与去重
    ↓
Rerank 重排序
    ↓
置信度判断
    ├── 足够 → Prompt 组装 → LLM 生成答案 → 引用溯源
    └── 不足 → 追问补充信息 / 无答案提示 / 转人工
    ↓
答案校验（防幻觉 / 风险提示 / 来源绑定）
    ↓
返回答案 + 引用来源
    ↓
日志记录
```

## 四、多路混合检索架构

这是本系统的核心技术亮点，采用四路并行召回 + 融合重排的策略。

```
用户问题
    │
    ├──→ [路由1] 向量语义检索
    │         └─ Embedding(问题) → MongoDB Vector Search → Top-K 语义相关 Chunk
    │
    ├──→ [路由2] 关键词精确检索
    │         └─ BM25(问题关键词) → 故障码 / 车型 / 部件 / 政策编号 精确匹配
    │
    ├──→ [路由3] 结构化业务查询
    │         └─ 车辆档案 / 保养记录 / 质保规则 → MongoDB 结构化查询
    │
    └──→ [路由4] 历史案例检索
              └─ 故障现象相似度 → 典型案例集合 → 相似案例 Top-K

各路结果 → 合并去重 → 元数据过滤 → Rerank 重排序 → Top-5 上下文
```

**四路检索的价值：**

| 检索路径 | 解决的问题 | 示例 |
|---|---|---|
| 向量语义 | 口语化描述、模糊表达 | "车子突然没劲" → 动力下降故障 |
| 关键词精确 | 专业术语、编号 | "P0A0F" → 精确故障码释义 |
| 结构化查询 | 个性化判断 | "我这辆车还在保吗" → 查车辆档案+质保规则 |
| 案例检索 | 经验复用 | "有没有类似案例" → 相似维修案例 |

## 六、质保预判逻辑设计

质保预判是"RAG + 规则引擎"的结合，不能单纯依赖 LLM 判断。

```python
def warranty_precheck(vehicle, component, fault_description):
    """
    质保预判逻辑：
    1. 查询车辆档案（购车日期、里程）
    2. 查询质保规则（年限、里程、适用部件）
    3. 查询保养记录（是否按时保养）
    4. 检查免责条件（事故、改装、泡水等）
    5. 综合判断并输出预判结果
    """
    rules = get_warranty_rules(vehicle.model, component)
    
    time_ok = check_time_warranty(vehicle.purchase_date, rules.years)
    mileage_ok = check_mileage_warranty(vehicle.mileage, rules.mileage_limit)
    maintenance_ok = check_maintenance_records(vehicle.id)
    exclusion_risk = check_exclusion_keywords(fault_description, rules.exclusions)
    
    if not time_ok or not mileage_ok:
        return PreCheckResult.LIKELY_OUT_OF_WARRANTY
    elif not maintenance_ok:
        return PreCheckResult.MANUAL_REVIEW_REQUIRED
    elif exclusion_risk:
        return PreCheckResult.MANUAL_REVIEW_REQUIRED
    else:
        return PreCheckResult.LIKELY_IN_WARRANTY
```

## 七、系统非功能需求

| 指标 | 目标值 | 说明 |
|---|---|---|
| 问答响应时间 | < 5秒 | 含检索 + LLM 生成 |
| 文档导入时间 | < 2分钟/份 | 50页以内文档 |
| 向量检索延迟 | < 200ms | Top-10 召回 |
| 系统可用性 | > 99.5% | 月度统计 |
| 并发用户数 | > 100 | 同时在线 |
| 数据安全 | 接口鉴权 + HTTPS | JWT Token |
| 幻觉控制 | 引用来源覆盖率 > 90% | 答案必须有资料依据 |
