## 二、关键选型理由

### 2.1 为什么选 Milvus 而不是 MongoDB Atlas Vector Search？

**原因一：混合搜索能力**

Milvus 原生支持稠密向量 + 稀疏向量的混合搜索，通过 RRF（Reciprocal Rank Fusion）融合两路结果，召回率更高。MongoDB Atlas Vector Search 虽然也支持向量检索，但混合搜索能力不如 Milvus 成熟。

**原因二：性能优势**

Milvus 专门为向量检索设计，支持 HNSW/IVF 等索引，查询性能优于 MongoDB 的向量扩展。

**原因三：老师项目经验**

原项目（掌柜智库）使用 Milvus，已验证稳定性和性能，可直接复用经验。

**选型对比：**

| 数据库 | 优势 | 劣势 | 本项目适合度 |
|--------|------|------|-------------|
| Milvus | 混合搜索强，性能优 | 独立部署，运维成本高 | ⭐⭐⭐⭐⭐ 推荐 |
| MongoDB Atlas | 一体化，运维简单 | 向量检索能力弱于Milvus | ⭐⭐⭐ 适合小规模 |
| Elasticsearch | 全文检索强 | 向量检索不如专业向量库 | ⭐⭐ 适合第二阶段 |

### 2.2 为什么选 MinerU 而不是 PyMuPDF/python-docx？

**原因一：一站式解析**

MinerU 支持 PDF、Word、图片等多种格式，无需为每种格式写不同的解析器。

**原因二：图片提取能力强**

MinerU 可以自动提取文档中的图片，并识别图片内容（配合视觉模型），这是 PyMuPDF 做不到的。

**原因三：老师项目经验**

原项目（掌柜智库）使用 MinerU，已验证稳定性和效果，可直接复用代码。

**选型对比：**

| 工具 | 优势 | 劣势 | 本项目适合度 |
|------|------|------|-------------|
| MinerU | 一站式解析，图片提取强 | 需要API调用 | ⭐⭐⭐⭐⭐ 推荐 |
| PyMuPDF | 轻量，本地运行 | 图片提取弱 | ⭐⭐⭐ 适合简单场景 |
| python-docx | Word解析好 | 只支持Word | ⭐⭐ 仅作备选 |

### 2.3 为什么选 BGE-M3 而不是其他Embedding模型？

**原因一：混合向量支持**

BGE-M3 同时生成稠密向量（1024维）和稀疏向量，配合 Milvus 混合搜索效果最好。

**原因二：中文语义理解强**

BAAI 开源，MTEB 中文排行榜表现优秀，适合商用车业务中文资料。

**原因三：本地部署**

可本地部署，保证企业数据安全，不依赖外部API。

**选型对比：**

| 模型 | 优势 | 劣势 | 本项目适合度 |
|------|------|------|-------------|
| BGE-M3 | 混合向量，中文强 | 需要GPU/CPU资源 | ⭐⭐⭐⭐⭐ 推荐 |
| text-embedding-3 | API调用简单 | 依赖外部API | ⭐⭐⭐ 适合快速验证 |
| M3E | 开源免费 | 效果不如BGE | ⭐⭐ 适合小项目 |

### 2.4 为什么选 mimo-v2.5-pro 而不是 Qwen/DeepSeek？

**原因一：兼容OpenAI协议**

mimo 兼容 OpenAI API 协议，可以直接用 LangChain 的 ChatOpenAI，无需额外适配。

**原因二：小米自研**

小米自研模型，有独立的API服务，稳定性有保障。

**原因三：成本优势**

相比 Qwen/DeepSeek，mimo 的 API 价格可能更有优势（具体需对比）。

**原因四：面试加分**

使用小米的模型，面试时可以说"我用的是小米的mimo模型"，展示技术视野。

**选型对比：**

| 模型 | 优势 | 劣势 | 本项目适合度 |
|------|------|------|-------------|
| mimo-v2.5-pro | 兼容OpenAI，小米自研 | 较新，社区小 | ⭐⭐⭐⭐⭐ 推荐 |
| qwen-plus | 阿里云成熟，社区大 | 依赖阿里云 | ⭐⭐⭐⭐ 适合生产 |
| DeepSeek | 推理能力强，价格低 | API稳定性待验证 | ⭐⭐⭐ 适合研究 |

### 2.5 为什么选 LangGraph 而不是 LangChain？

**原因一：图编排更灵活**

LangGraph 支持条件边、并发节点、循环等复杂流程，比 LangChain 的 Chain 更灵活。

**原因二：状态管理更好**

LangGraph 的 StateGraph 天然支持状态管理，每个节点可以读写 state，流程更清晰。

**原因三：老师项目经验**

原项目（掌柜智库）使用 LangGraph，已验证稳定性和效果，可直接复用代码。
