# 用户角色与权限设计

> 文档编号：DOC-02-01 | 版本：v1.0 | 更新时间：2026-06-12

---

## 一、角色总览

系统共设计 **6 类用户角色**，分为 C端（终端客户侧）和 B端（售后服务侧）两大类。

```
角色体系
├── C端（终端客户侧）
│   ├── 终端客户/司机         → 车主，实际使用车辆的人
│   └── 车队管理员            → 企业车队/物流公司负责人
│
└── B端（售后服务侧）
    ├── 售后服务顾问          → 服务站接待，处理报修单
    ├── 维修技师              → 负责实际检测和维修
    ├── 售后工程师            → 厂端技术支持，处理疑难
    └── 知识库管理员          → 上传、审核、发布知识文档
```

---

## 二、各角色详细说明

### 2.1 终端客户/司机（customer）

**典型身份**：商用车车主、司机、个体货运经营者

**使用场景**：
- 车辆日常使用中出现问题，需要快速判断是否严重
- 想了解车辆保养周期和保养项目
- 不确定某个故障是否属于质保范围
- 需要预约服务站或提交报修

**核心功能权限**：

| 功能模块 | 权限 | 说明 |
|---|---|---|
| 账号注册/登录 | ✅ | 手机号注册 |
| 绑定车辆 | ✅ | 通过 VIN 或车牌绑定 |
| 智能问答 | ✅ | 查询保养、质保、故障、维修知识 |
| 自助诊断 | ✅ | 描述故障，获取诊断建议 |
| 质保预判 | ✅ | 基于车辆档案和质保规则初步判断 |
| 保养提醒查询 | ✅ | 查询是否到达保养节点 |
| 提交报修单 | ✅ | 自助创建报修单 |
| 查看报修进度 | ✅ | 仅查看自己的报修单 |
| 服务评价 | ✅ | 对维修结果评分 |
| 上传文档 | ❌ | 无权上传知识文档 |
| 查看其他用户数据 | ❌ | 数据隔离 |

---

### 2.2 车队管理员（fleet_admin）

**典型身份**：物流公司、运输企业的车辆管理负责人

**使用场景**：
- 统一管理企业名下多辆商用车
- 查看车队整体保养状态和质保状态
- 追踪车队报修进度

**核心功能权限**：

| 功能模块 | 权限 | 说明 |
|---|---|---|
| 车队车辆管理 | ✅ | 可绑定和管理企业名下所有车辆 |
| 批量查询保养状态 | ✅ | 查看所有车辆保养情况 |
| 查看全部报修单 | ✅ | 查看企业名下所有车辆工单 |
| 智能问答 | ✅ | 同终端客户 |
| 提交报修单 | ✅ | 可代企业车辆提交 |
| 用户管理 | ❌ | 无法管理其他企业的车辆 |

---

### 2.3 售后服务顾问（service_advisor）

**典型身份**：经销商/服务站前台接待人员

**使用场景**：
- 接待客户时快速查询政策和流程
- 处理客户提交的报修单
- 解释质保条款和保养建议

**核心功能权限**：

| 功能模块 | 权限 | 说明 |
|---|---|---|
| 智能问答 | ✅ | 可查询所有发布知识 |
| 查看报修单 | ✅ | 查看本服务站工单 |
| 受理报修单 | ✅ | 接收并更新工单状态 |
| 查看 AI 诊断结果 | ✅ | 查看系统初步诊断 |
| 更新工单状态 | ✅ | 推进报修单流程 |
| 上传案例 | ✅ | 提交典型案例 |
| 上传知识文档 | ❌ | 无上传权限 |
| 发布/审核知识 | ❌ | 无权限 |

---

### 2.4 维修技师（technician）

**典型身份**：服务站实际维修操作人员

**使用场景**：
- 查询维修步骤和故障码含义
- 参考典型案例解决疑难问题
- 填写维修结论

**核心功能权限**：

| 功能模块 | 权限 | 说明 |
|---|---|---|
| 智能问答 | ✅ | 重点查询维修手册和案例 |
| 故障码查询 | ✅ | 精确查询故障码含义 |
| 案例检索 | ✅ | 查找相似历史维修案例 |
| 查看工单详情 | ✅ | 查看本服务站已分配工单 |
| 填写维修结论 | ✅ | 更新工单维修结果 |
| 提交案例 | ✅ | 将维修经验沉淀为案例 |

---

### 2.5 售后工程师（after_sales_engineer）

**典型身份**：厂端技术支持工程师

**使用场景**：
- 处理经销商升级的复杂疑难问题
- 审核典型案例
- 支持质保争议处理

**核心功能权限**：

| 功能模块 | 权限 | 说明 |
|---|---|---|
| 智能问答 | ✅ | 可查询所有知识 |
| 查看所有服务站工单 | ✅ | 跨服务站查看 |
| 处理疑难工单 | ✅ | 接受升级工单 |
| 审核典型案例 | ✅ | 审核经销商提交案例 |
| 上传/更新知识文档 | ✅ | 可上传技术资料 |
| 查看用户反馈分析 | ✅ | 分析高频问题 |

---

### 2.6 知识库管理员（knowledge_admin）

**典型身份**：后台运营或技术支持人员

**使用场景**：
- 维护知识库内容版本
- 审核和发布知识文档
- 分析知识缺口

**核心功能权限**：

| 功能模块 | 权限 | 说明 |
|---|---|---|
| 上传所有类型文档 | ✅ | PDF/Word/Excel/手册/案例 |
| 文档解析与发布 | ✅ | 管理导入任务和发布状态 |
| 审核发布知识 | ✅ | 控制知识库内容质量 |
| 下线过期知识 | ✅ | 管理版本有效期 |
| 查看问答日志 | ✅ | 全量日志访问 |
| 查看用户反馈 | ✅ | 分析反馈数据 |
| 知识缺口分析 | ✅ | 分析无答案问题 |
| 管理用户账号 | ✅ | 角色分配和账号管理 |

## 四、知识权限设计

RAG 检索时，系统根据用户角色过滤可访问的知识片段。

| 知识类型 | 可访问角色 |
|---|---|
| 保养手册（通用） | 全部角色 |
| 质保手册（通用） | 全部角色 |
| 维修手册（详细步骤） | 技师、工程师、管理员 |
| 典型案例（已审核） | 全部角色 |
| 典型案例（待审核） | 工程师、管理员 |
| 内部技术通报 | 工程师、管理员 |
| 经销商专属资料 | 服务顾问、技师、工程师、管理员 |
