# 查询流程总览

> 文档编号：DOC-QUERY-00 | 版本：v1.0 | 更新时间：2026-06-13
> 适用项目：商用车智能售后诊断与报修知识助手（RAG）
> 参考原项目：掌柜智库（ai_0302_knowledge_base）

---

## 一、模块目的

**接收用户提问，经过意图识别→实体抽取→多路检索→融合→重排序→置信度判断→生成答案，最终返回精准回答。**

## 四、与原项目差异

| 对比项 | 原项目（掌柜智库） | 改造后（商用车RAG） |
|--------|-------------------|-------------------|
| **意图识别** | 简单的item_name确认 | 6类意图分类（售前/售后/用车/投诉/闲聊/办理） |
| **实体抽取** | 无 | 抽取车型/VIN/故障码/里程/部件 |
| **检索方式** | 向量+HyDE+网络搜索 | 增加BM25关键词检索+案例检索+结构化查询 |
| **结果融合** | RRF融合 | RRF融合+元数据过滤 |
| **置信度** | 无 | 置信度判断+追问机制 |
| **答案格式** | 纯文本 | 文本+引用编号+风险提示 |
| **历史存储** | MongoDB对话历史 | 增加QA会话表+引用表 |
| **Prompt** | 通用Prompt | 按意图加载不同Prompt |

## 六、执行流程

```
1. 用户提问 → POST /query
2. node_intent_recognition → 识别意图（售前/售后/...）
3. node_entity_extraction → 抽取实体（车型/VIN/故障码）
4. 根据意图决定后续流程：
   - 售前/售后/用车 → 走检索流程
   - 投诉/闲聊 → 直接LLM回答（不走检索）
   - 业务办理 → 调用第三方API
5. node_search_embedding + node_search_embedding_hyde + node_web_search_mcp + node_keyword_search → 并行检索
6. node_rrf → 融合多路结果
7. node_rerank → 重排序
8. node_metadata_filter → 元数据过滤
9. node_confidence_check → 置信度判断
10. node_answer_output → 生成答案
11. node_save_qa → 保存QA记录
12. 返回答案给用户
```
