## 一、uv 包管理器

本项目使用 `uv` 作为 Python 包管理器（替代 pip + venv）。

### 1.1 核心命令

| 命令 | 作用 | 使用场景 |
|------|------|----------|
| `uv sync` | 同步依赖：根据 `pyproject.toml` 和 `uv.lock` 安装/更新所有依赖到 `.venv` | 每次拉取代码后、修改依赖后 |
| `uv add <包名>` | 添加依赖：自动更新 `pyproject.toml` + `uv.lock` + 安装 | 需要新增第三方库时 |
| `uv remove <包名>` | 移除依赖：自动从 `pyproject.toml` + `uv.lock` 删除 | 不再需要某个库时 |
| `uv lock` | 重新生成锁文件：不安装，只更新 `uv.lock` | 手动改了 `pyproject.toml` 后 |
| `uv run <命令>` | 在虚拟环境中运行命令 | 运行 Python 脚本、启动服务 |
| `uv python list` | 列出可用的 Python 版本 | 查看环境 |
| `uv venv` | 创建虚拟环境 | 首次初始化 |

### 1.2 关键文件

| 文件 | 作用 | 是否提交到 Git |
|------|------|---------------|
| `pyproject.toml` | 项目元数据 + 依赖声明（手动改） | ✅ 是 |
| `uv.lock` | 依赖锁定文件（自动生成，锁定精确版本） | ✅ 是 |
| `.venv/` | 虚拟环境目录（自动创建） | ❌ 否（已在 .gitignore） |

### 1.3 `uv sync` 详解

```bash
uv sync
```

**作用**：
1. 读取 `pyproject.toml` 中的 `dependencies` 列表
2. 读取 `uv.lock` 中锁定的精确版本
3. 对比 `.venv/` 中已安装的包
4. 安装缺失的包、更新版本不一致的包、卸载多余的包

**输出示例**：
```
Resolved 202 packages in 181ms    # 解析依赖树
Checked 182 packages in 7ms       # 检查已安装包
```

**使用场景**：
- 拉取代码后第一次运行
- 修改 `pyproject.toml` 添加/删除依赖后
- 切换分支后
- 协作开发时同步队友的依赖变更

### 1.4 `uv add` 详解

```bash
# 添加单个包
uv add fastapi

# 添加指定版本
uv add "fastapi>=0.100.0"

# 添加开发依赖（不会打包到生产环境）
uv add --dev pytest

# 添加多个包
uv add fastapi uvicorn pydantic
```

**作用**：
1. 将包名写入 `pyproject.toml` 的 `dependencies`
2. 解析依赖树，更新 `uv.lock`
3. 在 `.venv/` 中安装包

### 1.5 `uv run` 详解

```bash
# 运行 Python 脚本
uv run python app/main.py

# 运行模块
uv run python -m app.api.http.import_server

# 运行 pytest
uv run pytest tests/

# 运行自定义脚本
uv run python scripts/init_milvus_collections.py
```

**作用**：在 `.venv` 虚拟环境中执行命令，无需手动激活虚拟环境。

## 二、本项目的环境配置流程

### 2.1 首次搭建

```bash
# 1. 克隆代码
git clone <仓库地址>
cd auto-carcrm

# 2. 安装依赖
uv sync

# 3. 配置环境变量
cp .env.example .env
# 编辑 .env 填入 API Key、数据库地址等

# 4. 启动基础设施（Milvus + MongoDB + MinIO）
docker compose -f docker/docker-compose.yml up -d milvus mongo minio

# 5. 初始化集合和索引
uv run python scripts/init_milvus_collections.py
uv run python scripts/init_mongo_collections.py

# 6. 启动服务
uv run python -m app.api.http.import_server   # 导入服务（端口8000）
uv run python -m app.api.http.query_server     # 查询服务（端口8001）
```

### 2.2 日常开发

```bash
# 拉取最新代码后
git pull
uv sync                    # 同步依赖变更

# 添加新依赖后
uv add rank_bm25           # 自动更新 pyproject.toml + uv.lock
uv sync                    # 安装到 .venv

# 运行服务
uv run python -m app.api.http.query_server
```

### 2.3 依赖冲突处理

```bash
# 查看依赖树
uv tree

# 查看某个包的依赖来源
uv tree --invert --package fastapi

# 强制重新解析
uv lock --upgrade

# 清理缓存
uv cache clean
```

## 三、常见问题

### Q1: `uv sync` 报错 "No Python found"

```bash
# 安装 Python 3.12
uv python install 3.12

# 指定 Python 版本
uv sync --python 3.12
```

### Q2: 添加依赖后代码报错 "ModuleNotFoundError"

```bash
# 重新同步
uv sync

# 确认包已安装
uv pip list | grep <包名>
```

### Q3: `uv.lock` 冲突

```bash
# 删除锁文件重新生成
rm uv.lock
uv lock
uv sync
```

### Q4: 切换 Python 版本

```bash
# 方法1：修改 pyproject.toml
# requires-python = ">=3.11,<3.13"  →  ">=3.12,<3.13"

# 方法2：指定版本同步
uv sync --python 3.12
```

## 四、与 pip 的对比

| 操作 | pip | uv |
|------|-----|-----|
| 安装依赖 | `pip install -r requirements.txt` | `uv sync` |
| 添加依赖 | `pip install fastapi` + 手动改 requirements.txt | `uv add fastapi`（自动更新） |
| 虚拟环境 | `python -m venv .venv` + `source .venv/bin/activate` | 自动管理，无需手动激活 |
| 锁文件 | 无（需手动 pip freeze） | `uv.lock` 自动生成 |
| 速度 | 慢 | 快 10-100 倍 |
| 运行命令 | `.venv/bin/python app.py` | `uv run python app.py` |
