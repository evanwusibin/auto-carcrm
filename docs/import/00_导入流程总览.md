# 导入流程总览

> 文档编号：DOC-IMPORT-00 | 版本：v1.0 | 更新时间：2026-06-13
> 适用项目：商用车智能售后诊断与报修知识助手（RAG）
> 参考原项目：掌柜智库（ai_0302_knowledge_base）

---

## 一、模块目的

**将外部文档（PDF/MD）解析、切块、向量化后存入Milvus，同时将元信息存入MongoDB，支持后续的语义检索和文档管理。**

---

## 二、全景图

```
┌─────────────────────────────────────────────────────────────────────┐
│  导入流程（10个节点）                                                 │
└─────────────────────────────────────────────────────────────────────┘

输入：用户上传的文件（PDF/MD）
输出：数据存入 Milvus + 知识文档存入 MongoDB

原项目节点（7个，不改造）：
  ┌──────────┐   ┌──────────────┐   ┌──────────┐   ┌──────────────────┐
  │ 1.entry  │──→│ 2.pdf_to_md  │──→│ 3.md_img │──→│ 4.document_split │
  │ 文件识别 │   │ PDF转MD      │   │ 图片增强 │   │ 文档切块         │
  └──────────┘   └──────────────┘   └──────────┘   └──────────────────┘
                                                          │
                                                          ▼
  ┌──────────────┐   ┌──────────────┐   ┌──────────────┐
  │ 5.item_name  │──→│ 6.embedding  │──→│ 7.import_    │
  │ 主体识别     │   │ 向量化       │   │   milvus     │
  └──────────────┘   └──────────────┘   └──────────────┘
                                              │
新增节点（3个）：                               ▼
  ┌──────────────┐   ┌──────────────┐   ┌──────────────┐
  │ 8.doc_meta   │──→│ 9.save_      │──→│ 10.publish   │
  │ 元数据抽取   │   │   knowledge  │   │ 审核发布     │
  └──────────────┘   └──────────────┘   └──────────────┘
                                              │
                                              ▼
                                        导入完成 ✅
```

## 三、节点清单

| 序号 | 节点名 | 作用 | 对应service | 状态 |
|------|--------|------|-------------|------|

| 2 | node_pdf_to_md | PDF转MD | pdf_parse_service.py | 原项目 |
| 3 | node_md_img | 图片增强 | enrich_markdown_images.py | 原项目 |
| 4 | node_document_split | 文档切块 | split_service.py | 原项目 |
| 5 | node_item_name_recognition | 主体识别 | item_name_service.py | 原项目 |
| 6 | node_bge_embedding | 向量化 | embedding_service.py | 原项目 |
| 7 | node_import_milvus | 入库 | index_service.py | 原项目 |
| 8 | node_doc_meta | 元数据抽取 | doc_meta_service.py | 🆕 新增 |
| 9 | node_save_knowledge | 知识文档持久化 | knowledge_persist_service.py | 🆕 新增 |
| 10 | node_publish | 审核发布 | knowledge_persist_service.py | 🆕 新增 |

## 四、state字段变化

| 阶段 | 新增字段 | 类型 | 说明 |
|------|----------|------|------|
| entry | local_file_path, file_title, local_dir | str | 文件基本信息 |
| entry | is_md_read_enabled, is_pdf_read_enabled | bool | 路由开关 |
| pdf_to_md | md_path, md_content | str | MD文件信息 |
| md_img | md_path(更新), md_content(更新) | str | 图片增强后的MD |
| document_split | chunks | list[dict] | 切块列表 |
| item_name | item_name | str | 主体名称 |
| embedding | chunks(更新) | list[dict] | 加了向量的chunks |
| doc_meta 🆕 | vehicle_model, doc_version, effective_date, expiry_date, doc_type | str | 元数据 |
| save_knowledge 🆕 | knowledge_doc_id | str | MongoDB文档ID |
| publish 🆕 | import_status | str | "completed" |

---

## 五、chunks字段演变

| 阶段 | chunks新增字段 |
|------|----------------|
| document_split | content, title, parent_title, part, file_title |
| item_name | item_name |
| embedding | dense_vector, sparse_vector |

---

## 六、与原项目总体差异

| 维度 | 原项目 | 改造后 |
|------|--------|--------|
| 节点数量 | 7个 | 10个（新增3个） |
| 元数据 | 只有item_name | 增加车型/版本/来源/有效期/标签 |
| 持久化 | 只存Milvus | 额外存MongoDB知识文档表 |
| 状态管理 | 无 | draft→published |
| 审核机制 | 无 | 支持自动/人工发布 |
| 原有节点 | 不变 | 不变 |
