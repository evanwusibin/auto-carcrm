## 一、项目背景

### 1.1 业务背景

在商用车售后服务场景中，服务顾问、维修技师、售后工程师和客服人员需要频繁查询车辆保养规则、质保政策、维修流程、故障排查方法和典型案例处理方案。

传统售后服务存在以下核心痛点：

| 痛点 | 具体表现 | 影响 |
|---|---|---|
| **知识分散** | 保养手册、质保手册、维修手册分散在不同系统和PDF中 | 查询效率低，新人培训成本高 |
| **口径不统一** | 不同服务顾问对同一政策解释不一致 | 客户体验差，投诉率高 |
| **案例复用不足** | 历史典型故障处理方案无法有效传承 | 重复问题反复排查，效率低 |
| **用户自助能力弱** | 车辆出现异常时用户无法初步判断是否需要报修 | 无效报修多，服务站压力大 |
| **质保判断依赖经验** | 是否在保、是否因保养不当失效需要人工逐一判断 | 纠纷多，处理时间长 |

### 1.2 项目定位

本项目定位为 **比亚迪商用车售前售后一体化系统的核心AI能力模块**，第一阶段聚焦于售后场景，基于 RAG（检索增强生成）技术构建智能售后诊断与报修知识助手。

项目不是简单的聊天机器人，而是：
- 将企业非结构化知识转化为 **可检索、可问答、可追溯的知识资产**
- 将用户售后服务流程转化为 **自助诊断 → 智能报修 → 工单处理 → 案例沉淀** 的完整闭环
- 将大模型能力与 **业务规则、车辆档案、保养记录** 深度结合，实现有依据、可核验的智能服务

### 1.3 项目范围

**第一阶段（MVP）覆盖范围：**

```
数据范围：保养保修手册、质保手册、维修手册、典型案例处理方案
用户范围：车主/司机、车队管理员、售后服务顾问、维修技师、售后工程师、知识库管理员
核心能力：文档导入 → 知识解析 → 向量检索 → 智能问答 → 引用溯源 → 反馈优化
智能服务：自助故障诊断 + 质保预判 + 智能报修单生成
```

**暂不覆盖（后续阶段）：**
- 售前合同分析、投标书分析
- 销售拜访报告分析
- 签单、开单、回款、打卡等结构化业务问数

## 二、项目目标

### 2.1 业务目标

| 目标 | 描述 | 衡量指标 |
|---|---|---|
| 提升知识查询效率 | 售后人员通过自然语言查询，无需翻阅手册 | 平均查询时间 < 30秒 |
| 提升服务口径一致性 | 基于统一知识库回答，减少人为差异 | 答案来源可追溯比例 > 90% |
| 降低无效报修比例 | 用户通过自助诊断初步判断是否需要报修 | 无效报修减少 20% |
| 提升案例复用效率 | 历史典型案例可被检索和参考 | 案例入库数量持续增长 |
| 支持质保预判 | 系统自动结合车辆档案和质保规则给出初判 | 预判准确率 > 85% |

### 2.2 产品目标

1. **知识服务化**：将静态PDF手册转化为动态可问答的知识服务
2. **诊断智能化**：用户描述故障后，系统给出初步诊断和风险等级
3. **报修流程化**：从诊断到报修的流程无缝衔接，减少手动填写
4. **评估闭环化**：通过用户反馈持续优化知识库和检索效果

### 2.3 技术目标

1. 完成从文档导入到向量索引的完整知识入库链路
2. 实现多路混合检索（向量 + 关键词 + 结构化查询 + 案例检索）
3. 实现答案引用溯源，每条回答必须标注来源文档和章节
4. 控制模型幻觉：无资料依据时明确告知，不编造政策和参数
5. 支持多轮对话上下文管理

## 三、系统角色与服务对象

| 角色 | 典型身份 | 核心诉求 |
|---|---|---|
| **终端客户/司机** | 实际驾驶商用车的人 | 车辆故障时快速判断原因、是否需要报修、质保是否还在 |
| **车队管理员** | 物流公司、企业车队负责人 | 查看车队车辆状态、保养情况、报修记录 |
| **售后服务顾问** | 服务站接待人员 | 快速理解用户问题、判断是否建单、解释质保政策 |
| **维修技师** | 服务站维修人员 | 查询维修手册、故障码、典型案例、排查步骤 |
| **售后工程师** | 厂端技术支持 | 处理疑难问题、审核案例、支持经销商 |
| **知识库管理员** | 后台运营/技术支持 | 上传手册、维护案例、管理版本和状态 |

## 四、项目核心价值

### 4.1 对终端客户
- 车辆出现问题时，不需要立即打电话或跑服务站，可以先自助诊断
- 系统根据车辆档案和质保规则，给出质保预判，减少无效跑腿
- 智能生成报修单，减少手动填写，信息更完整

### 4.2 对经销商/服务站
- 接到报修单时，可以看到 AI 初步诊断结果，提前准备
- 维修知识查询更高效，技师不需要手翻手册
- 维修结论可以直接沉淀为典型案例，减少重复劳动

### 4.3 对厂家
- 知识库统一管理，版本清晰，避免多版本并存导致口径混乱
- 高频问题和无答案问题自动汇总，支持知识库持续优化
- 典型案例自动沉淀，形成企业知识资产

## 五、开发阶段规划

```
第一阶段（MVP）：售后知识导入 + RAG问答 + 引用溯源 + 智能诊断 + 报修单
第二阶段：典型案例辅助诊断 + 维修工单接入 + 经销商案例沉淀
第三阶段：售前销售问数 + 拜访分析 + 合同文件检索
第四阶段：售前售后一体化客户画像 + 复购机会分析
第五阶段：全业务链路 AI 智能助手
```
