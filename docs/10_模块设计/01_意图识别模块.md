## 一、模块概述

### 1.1 作用

意图识别模块负责分析用户问题，识别查询意图类型，为后续检索策略提供依据。

### 1.2 意图类型

| 意图类型 | 英文名称 | 说明 | 示例 |
|----------|----------|------|------|
| 售前咨询 | pre_sales_consultation | 产品参数、价格、配置 | "这款车多少钱？" |
| 售后服务 | after_sales_service | 维修、保养、故障 | "发动机异响怎么办？" |
| 用车指导 | vehicle_usage_guide | 操作方法、注意事项 | "怎么开空调？" |
| 投诉情绪 | complaint_emotion | 投诉、不满、情绪发泄 | "你们的车太差了！" |
| 业务办理 | business_transaction | 报修、预约、查询进度 | "我要报修" |
| 闲聊寒暄 | general_chat | 问候、闲聊 | "你好" |

## 三、核心代码解析

### 3.1 节点层 (node_intent_recognition.py)

```python
@node_log("node_intent_recognition")
def node_intent_recognition(state: QueryGraphState) -> QueryGraphState:
    """
    节点：意图识别
    
    输入：rewritten_query, item_names
    输出：intent（意图类型）
    """
    add_running_task(state.get("session_id", ""), "node_intent_recognition", state.get("is_stream", False))
    
    try:
        state = recognize_intent(state)
        logger.info(f"意图识别完成，意图类型：{state.get('intent', 'unknown')}")
    except Exception as e:
        logger.error(f"意图识别失败：{e}")
        state["intent"] = "after_sales_service"  # 默认意图
    
    add_done_task(state.get("session_id", ""), "node_intent_recognition", state.get("is_stream", False))
    return state
```

**关键点**：
1. 使用 `@node_log` 装饰器记录日志
2. 使用 `add_running_task` / `add_done_task` 记录任务状态
3. 异常时默认设为 `after_sales_service` 意图

### 3.2 服务层 (intent_recognition_service.py)

```python
def recognize_intent(state: QueryGraphState) -> QueryGraphState:
    """
    意图识别主函数
    
    输入：state（包含 rewritten_query）
    输出：state（新增 intent 字段）
    """
    # 1. 校验参数
    rewritten_query = validate_intent_state(state)
    
    # 2. 调用 LLM 识别意图
    intent_type = call_llm_recognize_intent(rewritten_query)
    
    # 3. 校验意图类型是否有效
    if intent_type not in INTENT_TYPES:
        logger.warning(f"识别到未知意图类型：{intent_type}，默认设为 after_sales_service")
        intent_type = "after_sales_service"
    
    # 4. 写入 state
    state["intent"] = intent_type
    
    return state
```

**关键点**：
1. 先校验参数，再调用 LLM
2. 校验意图类型是否有效
3. 无效意图默认设为 `after_sales_service`

**关键点**：
1. 使用 `{rewritten_query}` 占位符
2. 明确输出格式：只输出英文名称
3. 提供示例帮助 LLM 理解

## 五、常见问题

### Q1: 为什么默认意图是 after_sales_service？

**A**：因为本项目是商用车售后系统，大部分问题都是售后服务相关。当 LLM 无法识别意图时，默认设为售后服务最安全。

### Q2: 意图类型如何扩展？

**A**：修改 `intent_recognition_service.py` 中的 `INTENT_TYPES` 字典，并更新提示词中的意图类型定义。

### Q3: 意图识别失败怎么办？

**A**：节点层捕获异常后，默认设为 `after_sales_service` 意图，不会中断流程。
