## 五、节点详细说明

### 5.1 node_doc_meta（元数据抽取）

**作用**：从文档内容中抽取元数据

**输入**：
- `md_content`：文档内容
- `file_title`：文件标题

**输出**：
- `doc_type`：文档类型
- `vehicle_model`：车型
- `component`：部件
- `version`：版本号
- `effective_date`：生效日期
- `expire_date`：失效日期
- `visible_roles`：可见角色列表

**核心逻辑**：
```python
def extract_doc_meta(state):
    # 1. 校验参数
    md_content, file_title = validate_doc_meta_state(state)
    
    # 2. 调用 LLM 抽取元数据
    meta = call_llm_extract_doc_meta(md_content, file_title)
    
    # 3. 写入 state
    state["doc_type"] = meta.get("doc_type", "faq")
    state["vehicle_model"] = meta.get("vehicle_model", "")
    # ... 其他字段
    return state
```

---

### 5.2 node_save_knowledge（知识文档保存）

**作用**：将知识文档和 Chunk 保存到 MongoDB

**输入**：
- `chunks`：切块列表
- `item_name`：商品名称
- `file_title`：文件标题
- `doc_type`：文档类型
- `vehicle_model`：车型

**输出**：
- `doc_id`：文档 ID

**核心逻辑**：
```python
def save_knowledge(state):
    # 1. 校验参数
    chunks, item_name, file_title = validate_knowledge_state(state)
    
    # 2. 生成文档 ID
    doc_id = f"doc_{state.get('task_id', 'unknown')}"
    
    # 3. 保存知识文档
    save_knowledge_document(state, doc_id)
    
    # 4. 保存知识 Chunk
    save_knowledge_chunks(state, doc_id)
    
    # 5. 保存导入任务
    save_import_task(state, doc_id)
    
    # 6. 写入 state
    state["doc_id"] = doc_id
    return state
```

---

### 5.3 node_publish（审核发布）

**作用**：将文档状态设置为待审核

**输入**：
- `doc_id`：文档 ID
